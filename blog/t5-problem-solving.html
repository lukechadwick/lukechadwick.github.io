Technical Blog Part 5

Tell your non-tech friend a story about a time you:

2 July 2018

Got blocked on a simple problem
Solved a problem in an elegant way, In both cases:
What was the problem?
I was using C# to write an application for a game that reads memory from the game executable and performs a few tasks based on this, however the application was crashing.
What problem solving techniques did you use?
First I tried double checking my code but couldn't see any obvious problems. After this I used a debugger which allowed me go through the code step-by-step to see where the data was going and what was happening to it. This allowed me to locate and solve the error (Without going into too much detail the function I had that calculated a memory address was incorrect).
How did you feel throughout the process?
Debugging can be quite stressful or time consuming, but I had done some before so not too bad.
What did you learn?
Debuggers are a great way to take another look at your code and identify mistakes you might not see from simpler methods like looking at the console output or reading the source code.
